(1) Коротко опишите, какие задачи с помощью механизма ветвления решаются в GIT? Зачем выполняется их слияние?
Ветки нужны, чтобы несколько программистов могли вести работу над одним и тем же проектом, при этом не мешая друг другу.
Кроме того, ветки используются для тестирования экспериментальных функций или исправления багов.
Если эксперимент удался или нужно зафиксировать исправление бага, изменения с экспериментальной ветки переносятся 
на основную, если нет – новая ветка удаляется, а проект остается нетронутым.


(2) Укажите команды для:

переименования веток - git branch -m oldBranchName newBranchName; 

создания и переключения на новую ветку:
 - git branch branchName - создать новую ветку;
 - git checkout -b branchName - создать и сразу переключиться на новую ветку;
 - git checkout branchName - переключиться на нужную веткуж 

удаление ветки:
- git branch -d branchName;  
- git branch -D branchName – принудительно удалить ветку;
При удалении ветки нельзя находиться на данной ветке.

слияния веток:
для слияния необходимо переключиться на ветку, в которую нужно включить изменения, и выполнить команду 
- git merge branchName(название ветки с которой выполняется слияние).

1. Зачем нужен GIT и какие задачи решают с его помощью?  

Git - система контроля версий (консольная программа), для отслеживания и ведения истории изменения файлов, в проекте.
С помощью Git-a можно откатить свой проект до более старой версии, сравнивать, анализировать или отправить изменения в репозиторий.

2. Что такое репозиторий GIT и где он находится?

Репозиторий - хранилище файлов (кода) и истории их изменений. 
Git работает локально и репозитории хранятся в определенных папках (скрытая папка .git, для каждого проекта
отдельная папка) на жестком диске.

3. Для чего используется файл .gitignore?

.gitignore нужен для скрытия файлов и папок от системы контроля версий Git. Данный файл указывает Git-у, какие файлы нужно пропустить,
игнорировать и не добавлять в репозиторий. 

4. Что такое "коммит"? 

commit - снимок состояния или контрольная точка проекта (то есть коммит это копия проекта в момент времени, когда этот коммит был 
сделан)


Укажите базовые команды в рамках работы с GIT:


1. Настройка аккаунта локального пользователя.

git config --global -l  -проверить глобальные настройки
git config –global user.name “name” - настройка имени пользователя
git config –global user.email “email” -настройка электронной почты (указать почту как на GH)
git config --global core.editor 'название редактора' – привязка редактора к git


2. Создание репозитория.

git init - создать новый репозиторий

3. Добавление файлов в индекс и выполнение коммита.

git add . - добавить в очередь на коммит все файлы
git add имя файла - добавить в очередь на коммит определённый файл
git commit -m 'комментарий к комиту' - выполнение коммита

4. Просмотр журнала коммитов.
git log – посмотреть список комитов

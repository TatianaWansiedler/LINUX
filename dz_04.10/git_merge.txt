Укажите след. команды:

для полной очистки индекса - git reset 

для просмотра разницы между раб.каталогом и посл. коммитом - git diff 

для создания новой ветки и одновременного переключения в на нее - git checkout -b branchName

для сравнения веток между собой -  git diff ..branchName  (или git diff branchName..branchName)

для удаления локальной ветки - git branch -d branchName (необходимо переключиться с ветки которую нужно удалить)

для слияния веток - git merge branchName (необходимо переключиться на ветку в которую происходит слияние)

для перезаписи посл/коммита. - git commit --amend 


Коротко опишите причину возникновения конфликтов при слиянии и порядок действий для их решения.
- Обычно конфликт возникает при слиянии текущей локальной ветки и ветки, с которой выполняется слияние, 
когда изменяют одни и те же строки в файле.
- В таких случаях Git не может автоматически определить, какое изменение является правильным.
- Git помечает файл как конфликтующий и останавливает процесс слияния.
- В конфликтующем файле появятся дополнительные строки в месте где возник конфликт. 
- Что бы его решить необходимо в любом текстовом редакторе открыть файл и отредактировать его, оставив нужные строки кода.
- После редактирования файла выполнить команды git add и git commit для завершения слияния и создания нового коммита.
